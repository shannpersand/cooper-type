The Interface

➊ ➋ ➌ ➍ ➎ ➏
  1. Preview of the glyph selected in the glyph list.
  2. Glyph list (✓ has layers) of all glyphs in the font. Double click to show a glyph in the current Glyph Window.
  3. List of layers in the selected glyph from the glyph list. Delete selected layer by hitting the backspace key. Add a layer by pressing the ➕ button. The layer glyph will be the glyph currently selected in the Font Window, and the currently selected palette color will be assigned to it.
  4. Switch current palette.
  5. Colors in the current palette. Select a color to preview it in the color box below. Click on the color box to change the color. Add a color by pressing the ➕ button, delete selected color by pressing the backspace key.
  6. Foreground/Background color. These are used in the Preview area only and are not exported to the font.

Buttons

  • Select glyphs with layers – In the current Font Window, select all glyphs that have layers.
  • Mix palette – Fill current palette with random colors.
  • Export PNG – Save a PNG file of the current glyph.
  • Settings... – Display a drawer with more options.
  • Auto layers – Automatically assign layers for all glyphs.
  • Import font – Import color data from a font. Only the Microsoft color tables (COLR/CPAL) are imported.
  • Export font – Export color data to an existing font. The target font must be generated beforehand via RoboFont. The Export font button always exports to the last font that was generated from the current UFO. The font file is not being overwritten, but copied with a suffix indicating the color data format. Check the Settings drawer for export options.

Settings

Pressing the Settings ... button will open a drawer where you can set options in more detail.

➊ ➋ ➌ ➍
  1. Generate formats lets you choose which color formats should be exported. ➔ System support
  2. Auto layer suffix regex contains the regular expression that is used for the Auto layers function. ➔ Assigning layers automatically
  3. Generate bitmap sizes contains a comma-separated list of sizes to include for the bitmap-based color font formats (Mac OS/iOS/Google). Be aware that the bitmap rasterization is very slow.
  4. Prefer placed images over outlines – Use available bitmap images in the UFO instead of layer glyphs (not implemented yet).

Buttons

  • Test – Tests the current Auto layer suffix regex by selecting all glyphs that match it in the current Font Window.
  • Debug – Print debug information about the internal color data model to the Output Window.
  • Reset – Delete all color data from the current UFO.
