Assigning layers automatically

You can quickly collect all the color layers for each base glyph by using the Auto Layers function.

For this to work, you must use glyph name suffixes systematically to identify the different color layers.

Layer glyph names are identified by a Python-style regular expression (regex) which can be changed in the Settings drawer.

For example, if you have a glyph called circle, the default regex \.alt[0-9]{3}$ will match all glyphs called circle.alt000, circle.alt001, circle.alt002, etc.

If you want the base glyph to be included in the layer glyph sequence, check the option «Auto layers include base glyph» in the Settings drawer. When active, the base glyph will be included as the last (topmost) layer in the color glyph.

When using the Auto Layers function, the glyph palette will be filled with as many random colors as necessary.

Auto layer suffix regex

The Auto layer suffix regex in the Settings drawer determines which glyph name suffix will identify a matching layer glyph for a given base glyph.

The checkbox to the right of the Auto layer suffix regex textbox indicates if the current regex is valid, and will update its status as you type. Use the Test button to select all glyphs in the font window that match the current regex.

The default regex explained:

\.alt[0-9]{3}$

\. alt [0-9] {3} $
A period (period is a special character in a regex so it has to be escaped with a backslash) The letters «alt» Any number between 0 and 9 The previous expression must be present 3 times Matches the end of the glyph name

E. g. if you use the glyph name suffixes .layer01, .layer02, etc., your regex must be «\.layer[0-9]{2}$».
